# ADR-XXXX: Short title of the decision

- **Status:** Proposed <!-- Proposed | Accepted | Superseded by ADR-XXXX | Deprecated -->
- **Date:** YYYY-MM-DD
- **Owners:** who is accountable for this decision

## Context

What is the situation and the forces at play? What problem or constraint prompts a
decision now? Keep it factual — describe the world as it is, not the choice yet.

## Decision

The choice that was made, stated plainly and in the active voice ("We will ...").
Include enough detail that a reader can act on it without asking follow-ups.

## Alternatives

The other options considered, and why each was not chosen. "Do nothing" is a valid
alternative worth recording.

## Consequences

What becomes true because of this decision — both the benefits and the costs, new
obligations, and follow-up work it creates. Be honest about the downsides.

---

**Usage:** Copy this file to `NNNN-title.md` where `NNNN` is the next unused
four-digit number (this repo's ADRs start at `0001`). Fill in every section. ADRs
are an append-only log: never delete or rewrite an old ADR — if a decision changes,
write a new ADR and set the old one's status to `Superseded by ADR-XXXX`.

Note: `docs/` also holds design specs (for example `docs/workbench-dc-parity/`)
that are NOT ADRs and are not part of this numbering.
